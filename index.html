CLO 3AProblem Definition3 / 3 marks
🔬 Tab: Visualizer — left panel visible
👆 Point to the Edge list input box (top-left, labelled "Edge list (one u v per line)") and the k = field directly below it. These are the two inputs of the problem.
The input is an undirected graph G = (V, E) — vertices are items, edges are conflicts — and an integer k. In my tool, the graph is entered here as an edge list, one edge per line, and this field is k. The decision question is: does this graph contain an independent set of size at least k?
👆 Point to the Graph Visualization panel on the right — the circular canvas showing coloured vertices and edges.
An independent set is a subset of vertices where no two selected vertices share an edge. The output is either a valid independent set — shown here as selected vertices — or a conclusion that none of size k was found. The constraint is simple: if two vertices are connected, they cannot both be selected. This is useful for scheduling conflict-free tasks, allocating non-interfering resources, or choosing non-clashing courses.

CLO 4ABrute-Force Algorithm4 / 4 marks
🔬 Tab: Visualizer → Algorithm panel → click Brute Force
👆 In the left panel under "Algorithm", click Brute Force. Then click Preset: Triangle K₃. Press ⚡ Build Steps.
The first algorithm is the exact brute-force. It is guaranteed to find the maximum independent set because it checks every possible subset of vertices. For a graph with n vertices, there are 2ⁿ possible subsets. For each one it checks two conditions: is the size at least k, and are all selected vertices non-adjacent?
👆 Press ▶| Step slowly. Point to the Step-by-Step Log below the graph canvas — it explains each decision in plain English.
If any two selected vertices share an edge, the subset is rejected. If it passes, it is recorded as a valid candidate. The correctness argument: the optimal set S* is one of the 2ⁿ subsets enumerated — brute force must eventually reach it, accept it, and return it as the answer.
Complexity
2ⁿ subsets, each requiring up to O(n²) pair comparisons. Worst-case total: O(2ⁿ · n²).
👆 Point to the ⚠️ DISABLED — TOO LARGE warning (appears when graph exceeds 18 vertices — load Petersen to trigger it). Then point to the Complexity Growth chart at the bottom of the Visualizer — it shows the exponential vs polynomial curves.
Brute force is feasible only for small graphs — my tool disables it above 18 vertices. That warning itself demonstrates the boundary between the feasible and intractable regimes. The growth chart makes the gap visual: brute force shoots up exponentially while greedy stays flat.
💡 Demo three presets in order below — one sentence each. This proves the algorithm is general, not hardcoded.
👆 Click: Triangle K₃ (dense — IS size 1) → Path P₆ (sparse — larger IS) → Cycle C₅ (32 subsets, IS size 2). Run brute force on each.
In the Triangle K₃ every vertex is adjacent so the maximum independent set is size 1 — confirmed. On Path P₆ the graph is sparse so brute force finds a larger set. On Cycle C₅ it checks all 32 subsets and correctly returns size 2. Each example confirms the algorithm works on varied inputs.
CLO 3BGreedy Algorithm — Paradigm & Validity4 / 4 marks
🔬 Tab: Visualizer → Algorithm panel → click Greedy Heuristic
👆 Click Greedy Heuristic in the Algorithm panel. Load Preset: Cycle C₅. Press ⚡ Build Steps then step through slowly.
The second algorithm follows the greedy paradigm — a design strategy where we make the locally optimal choice at each step without reconsidering previous decisions. The greedy rule here is: always pick the vertex with the minimum current degree in the remaining graph, add it to the independent set, then remove it and all its neighbors.
👆 Point to the color legend on the graph panel: In IS (green selected vertex), Excluded (red neighbors), Removed (no longer in graph). Then point to the 🎯 How We Picked box in the Live Statistics panel — it updates with the reason for each choice.
The visualization shows this: after selecting a vertex it turns green, its neighbors turn red and are excluded, and the remaining graph shrinks. The "How We Picked" panel gives the plain-English reason for each step.
Validity Invariant
At every step, the selected set contains no edge between any two members. When vertex v is added, all neighbors are immediately removed — so no future pick can be adjacent to v. By induction the output is always a legal independent set.
However, greedy is not guaranteed to find the maximum independent set — it makes local choices, and a local choice can block a better global solution. If greedy finds an independent set of size at least k, we can safely answer YES because we have a concrete witness. But if greedy fails to find size k, we cannot conclude NO — a different selection might still succeed.
💡 That last sentence is critical. Slow down and say it clearly — it directly targets the rubric.
CLO 3CHeuristic Quality & Counterexample3 / 3 marks
🔬 Tab: Visualizer → load ⚠️ Greedy Fails preset
👆 Click Preset: ⚠️ Greedy Fails. Confirm Greedy Heuristic is selected. Press ⚡ Build Steps then step through slowly — one step at a time.
For Maximum Independent Set, no constant-factor polynomial-time approximation exists unless P = NP — proven by Håstad in 1999. So I do not claim a guaranteed approximation ratio. I analyze greedy as a heuristic. This preset is my live counterexample.
👆 Point to vertex 0 being selected first, then the excluded neighbors turning red — specifically vertices 1 and 2 disappearing from consideration.
Vertices 0, 1, 2, and 4 all have minimum degree 2 — a four-way tie. Greedy breaks ties by lowest index and picks vertex 0. Removing 0 eliminates vertices 1 and 2, which are part of the optimal solution. The remaining subgraph forces greedy to produce an independent set of size 2.
👆 Switch to Brute Force in the Algorithm panel. Press ⚡ Build Steps → ▶ Play. Point to the IS Size result shown in Live Statistics.
Brute force verifies the optimal independent set is size 3 — specifically {1, 2, 4}. Greedy is valid but not optimal. For k = 2 greedy succeeds; for k = 3 greedy fails even though a valid set of size 3 exists.

The heuristic works best on sparse or simple graphs like paths and cycles. It fails on adversarial graphs where the first local choice removes vertices needed for the global optimum.

CLO 5ANP-Completeness — NP Membership & Reduction10 / 10 marks
🔐 Switch to the NP-Completeness tab → you will see the interactive numbered proof
💡 This is worth the most marks. Take a breath before you start. Speak at half speed. Every sentence counts here.
👆 Navigate to the 🔐 NP-Completeness tab. The page shows a numbered interactive proof. Make sure you are viewing Step 1 — Independent Set ∈ NP (the first step card).
Now I will prove Independent Set is NP-complete. A problem is NP-complete if two things hold: it is in NP, and it is NP-hard.

Part 1 — Independent Set is in NP. The certificate is a proposed subset S ⊆ V. The verifier receives the graph, the integer k, and this set S. It runs two checks: first, whether |S| ≥ k — takes O(n). Second, whether every pair inside S has no edge — takes O(n²). Total verification time is O(n²) — polynomial. Therefore Independent Set ∈ NP.
👆 Press Next → to move to Step 2 — Reduction: Vertex Cover ≤ₚ Independent Set. Point to the Reduction Diagram — the box with arrows showing the mapping from VC to IS.
Part 2 — NP-Hardness. I reduce from Vertex Cover, a known NP-complete problem. Vertex Cover asks: does graph G have a set C ⊆ V of size at most k where every edge has at least one endpoint in C?
👆 Point to the Formal Mapping box labelled: f(G, k) = (G, |V| − k). This is visible in the Step 2 construction section.
The reduction is elegant in its simplicity: given a Vertex Cover instance (G, k), I output the Independent Set instance (G, |V| − k). The graph stays exactly the same — no new vertices, no new edges, no gadgets. Only the parameter changes from k to |V| − k.
Polynomial-Time Construction
Copy the graph: O(|V| + |E|). Compute |V| − k: one subtraction, O(1). Total: O(|V| + |E|) — linear in input size, clearly polynomial.
👆 Scroll down to the ★ Interactive Reduction Example at the bottom of Step 2. It has a Vertex Cover k input on the left and the derived IS k′ = |V| − k on the right. The default shows |V|=6, k=2, k′=4.
For example, with 6 vertices and Vertex Cover target k = 2, the reduction gives Independent Set target k' = 6 − 2 = 4. Same graph — just a new parameter.
CLO 5BReduction Correctness — Both Directions10 / 10 marks
🔐 Stay on NP-Completeness tab → Step 2 → scroll to sections 2b and 2c
👆 Point to the "Core Claim of the Reduction" box — the large ⟺ biconditional statement displayed prominently in Step 2.
Now I prove the reduction correct in both directions. The claim is: G has a vertex cover of size at most k  ⟺  G has an independent set of size at least |V| − k.
👆 Scroll to section 2b — "VC YES means IS YES".
Forward direction. Suppose Vertex Cover is YES — so there exists C ⊆ V with |C| ≤ k and every edge has an endpoint in C. Define S = V ∖ C.

Suppose S is not independent — then two vertices u, v ∈ S share an edge. But both are outside C, so that edge has no endpoint in C. This contradicts C being a vertex cover. Therefore S must be independent. Its size: |S| = |V| − |C| ≥ |V| − k. Independent Set is YES. ⟹ proved.
👆 Scroll to section 2c — "IS YES means VC YES".
Backward direction. Suppose Independent Set is YES — so there exists S ⊆ V with |S| ≥ |V| − k and no two vertices in S share an edge. Define C = V ∖ S.

Take any edge (u, v) in the graph. Since S is an independent set by assumption, u and v cannot both be in S. So at least one must lie in V ∖ S = C. This holds for every edge — so C is a valid vertex cover. Its size: |C| = |V| − |S| ≤ k. Vertex Cover is YES. ⟸ proved.
NP-Completeness Conclusion
The reduction is correct in both directions and runs in polynomial time. Since Vertex Cover is NP-complete and reduces to Independent Set, Independent Set is NP-hard. Since it is also in NP — Independent Set is NP-complete. ∎
👆 Scroll to the Conclusion box at the very bottom of the NP-Completeness page — it shows the full logical chain: Step 1 (NP) + Step 2 (NP-hard) → NP-complete.
Edge cases my tool handles: if k = 0 the answer is always YES — the empty set is independent. If k > |V| the answer is NO. If the graph has no edges, all vertices form an independent set. If the graph is complete, the maximum independent set has size 1.
